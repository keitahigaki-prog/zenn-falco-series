# 付録E: 本番導入前チェックリスト

Falco を本番環境に導入する際の確認事項を、ルール設計・運用・インフラ・組織の4つの観点からまとめたチェックリストです。

## 1. ルール設計面

### 1.1 どのルールを有効にするかの決定

- [ ] **セキュリティポリシーとの整合性確認**
  - [ ] 組織のセキュリティポリシーを再確認
  - [ ] コンプライアンス要件（PCI-DSS, HIPAA, SOC 2 など）を確認
  - [ ] リスク評価済みのルールを列挙
  - [ ] 監査要件の確認（ログ保持期間など）

- [ ] **ルール選択の基準化**
  - [ ] 有効にする upstream ルール（デフォルトセット）を明記
  - [ ] ルール有効化の意思決定プロセスを文書化
  - [ ] リスクレベル別（critical, high, medium, low）に分類
  - [ ] 本番環境で必ず有効にするルールを指定

- [ ] **環境別ルールの使い分け**
  - [ ] 開発環境用ルール設定
  - [ ] ステージング環境用ルール設定
  - [ ] 本番環境用ルール設定

### 1.2 例外（ホワイトリスト）の徹底

- [ ] **プロセス例外の完全化**
  - [ ] allowed_processes マクロを確認
  - [ ] 本番環境で実行されるすべてのプロセスを網羅
  - [ ] 定期実行タスク（cron）の登録確認

- [ ] **ユーザー例外の確認**
  - [ ] allowed_users マクロの確認
  - [ ] サービスアカウント（root, www-data など）の登録

本書では、さらに詳細なチェックリストを掲載しており、本番導入時のリスク低減に役立ちます。
