# 付録D: ルールレビュー観点チェックリスト

Falco ルールおよび例外の PR レビューで使うチェックリストです。PR テンプレートに貼り付けて運用してください。

## 共通

- [ ] 変更の目的が PR 説明文に書かれている
- [ ] 変更対象が upstream ルールか自作ルールかが明確
- [ ] upstream ルールを触る場合、`override` か `copy` かが明示されている

## ルール追加・変更時

- [ ] `condition` が一行ずつ読める粒度に分解されている（必要ならマクロ化）
- [ ] `output` に調査に必要なフィールド（`k8s.ns.name` / `container.image.repository` / `proc.pname` / `proc.cmdline` など）が含まれている
- [ ] `priority` が運用フロー（通知先・エスカレーション）と整合している
- [ ] 既存のマクロ・リストで表現できないかを検討済み
- [ ] テスト環境で発火・非発火の両方を確認している

## 例外追加時

- [ ] 例外の `name` が意味のある識別子になっている
- [ ] `REASON` / `OWNER` / `EXPIRES` がコメントで残されている
- [ ] 粒度が「process × 親プロセス × image」以上に狭い、または広い理由が説明されている
- [ ] この例外を通り抜けうる攻撃パターンが検討され、他ルールで backstop されている
- [ ] 時限例外の場合、棚卸しの仕組み（grep / CI チェック）に載っている

このチェックリストを運用に組み込むことで、ルール品質を一定に保ちながら、チーム内での一貫性を実現できます。
