# はじめに

Falcoを導入すると、多くのチームが同じ壁にぶつかります。

- アラートが多すぎて追えない
- どこまでチューニングしていいかわからない
- 例外を入れると「ザル」になりそうで怖い
- upstreamルールをどう扱えばいいかわからない
- update後に挙動が変わり、何を直せばよいか判断できない

Falcoは「導入」よりも、「運用」のほうが圧倒的に難しいツールです。前作『Falco実践シリーズ』では、Falcoを導入し、最初のルールを動かすところまでを解説しました。しかし現場で実際に詰まるのは、そのあとに続く長い運用フェーズです。

本書はその中でも特にハードルとなる **ルールチューニング** にフォーカスし、次の論点を体系的に解説します。

- condition の正しい読み方・分解の仕方
- フィールドの実務上の使い分け
- 例外設計の考え方と「ザル化」を避ける思想
- 誤検知の調査フロー（何から見るか、何で判断するか）
- upstream ルールの扱い方（override / copy / disable）
- 本番運用に耐えるルール設計と組織運用

単なる構文解説ではなく、**実際の運用でどう判断するか**、**どこまで触ってよいか** に踏み込んだ内容になっています。

Falcoを「使える状態」から「使い続けられる状態」へ引き上げるための一冊です。

---

## 本書の核となる価値

よくある本が「どう書くか」を説明するのに対し、本書は次の問いに答えることを目的としています。

- このアラートは消していいのか
- 例外はどの粒度で切るべきか
- upstream の更新をどう取り込むべきか
- 開発チームとどう責任分担するべきか
- 本番でどの順番で有効化していくべきか

つまり本書は、**ルールを「書く本」ではなく、ルールを「扱う本」** です。

## 想定読者

主ターゲット：

- Kubernetes / コンテナを運用している SRE / Platform Engineer
- クラウドセキュリティ担当
- Falco を導入したが、ルール調整で止まっている人

副ターゲット：

- OSS の Falco をこれから触る人
- ルール設計のレビューを任される人
- マネージド製品経由で Falco 系検知に触れている人

前提知識：

- Kubernetes とコンテナの基本操作
- Linux の基本（プロセス、ファイルディスクリプタ、シスコール）
- YAML の読み書き
- 前作『Falco実践シリーズ』相当の Falco 基礎

---

## 読者の変化（Before / After）

**Before**

- ルールが読めない
- 誤検知が多くて追えない
- 触るのが怖くて disable しか選べない

**After**

- 複雑な condition を一行ずつ分解して読める
- 誤検知を「狭く」潰せる
- どこまで触ってよいかの判断基準を持てる
- 本番反映の順序と手戻り対応を設計できる

---

## 執筆方針

1. **節ごとのフォーマットを統一**：「これは何か → いつ使うか → 具体例 → よくある失敗 → 実運用のポイント」
2. **構文説明で終わらせない**：必ず「判断基準」まで踏み込む
3. **upstream ルールをそのまま題材にする**：架空の例ではなく、読者がリポジトリで見られるものを使う
4. **図解より分解**：複雑な condition を一行ずつ日本語化する
5. **Falco OSS のみ**：Sysdig 製品固有の機能には触れない
